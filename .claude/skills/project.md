# 프로젝트 기록 작성

프로젝트별 상세 기록을 작성합니다.

## Triggers

다음과 같은 요청에 반응:
- "프로젝트 기록 작성해줘"
- "XX 프로젝트 문서화해줘"
- "프로젝트 회고 정리"
- "이번 프로젝트 기록 남겨줘"

## 질문 항목

### 기본 정보
1. "프로젝트명과 진행 기간은?"
2. "프로젝트의 목적과 배경은?"
3. "팀 구성은 어떻게 되었나요? (PM, 개발자, 디자이너 등)"

### 본인 역할
4. "본인이 담당한 영역은?"
5. "주요 기여 사항을 구체적으로 알려주세요. (작업, 상세 내용, 기여도)"

### 기술
6. "사용한 기술 스택은? (Frontend, Backend, Infra 등)"

### 성과
7. "정량적 성과가 있나요? (개선율, 성능 지표 등 Before/After)"
8. "정성적 성과는?"

### 도전과 해결
9. "기술적으로 어려웠던 점은?"
10. "어떻게 해결했나요?"

### 회고
11. "잘한 점, 아쉬운 점, 배운 점은?"
12. "관련 PR, 문서, 발표 자료 링크가 있나요?"

## 작성 가이드

- **기여도 명확히**: "개발" → "프론트엔드 개발 80%, API 설계 참여"
- **성과 정량화**: Before/After 수치 비교
- **기술적 도전**: 문제-해결-결과 구조로 서술
- **회고 균형**: 잘한 점과 개선점 모두 기록

## File Structure

```
projects/[project-name].md
```

파일명 규칙: 소문자, 하이픈(-) 사용
예: `goods-detail-renewal.md`, `checkout-optimization.md`
